# Een toegankelijk formulier begint met goede labels

Maak je formulier meer toegankelijk door het `<label>` element op de juiste manier in te zetten.

Het `<label>` element is niet alleen bedoeld om de bedoeling van de inhoud te beschrijven, maar er zit ook ingebouwde toegankelijkheid in die je met geen enkel ander element zo makkelijk kan bereiken. Elk zichtbaar formulier element moet daarom een bijbehorend `<label>` element hebben.
Voor mensen die het formulier gebruiken zonder de zichtbare pagina context is het invullen van elk formulierveld waar een gekoppeld label aan zit stukken makkelijker.

Ook kun je op het label klikken om automatisch het juiste formulierveld te selecteren.

## Fout: Helemaal geen label

Zonder label is het voor niemand duidelijk

## Fout: Placeholder in plaats van label

Met alleen een placeholder is het niet meer duidelijk waar de input voor is als de gebruiker begint met invullen
Ook lezen niet alle screen-readers de placeholder voor waardoor het net zo slecht werkt als zonder label

<Canvas>
  <Story name="Fout: Placeholder in plaats van label">
    <form className="utrecht-form">
      <input
        type="text"
        name="first_name"
        className="utrecht-textbox utrecht-textbox--html-input utrecht-form-field__input"
        placeholder="Voornaam"
      />
    </form>
  </Story>
</Canvas>

## Fout: Verkeede HTML element voor het label

Door geen `<label>` element te gebruiken mis je de ingebouwde logica die zorgt dat hij als label kan worden geinterpreteerd.
Er bestaat zo geen enkele koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

<Canvas>
  <Story name="Fout: Verkeerd element als label">
    <form className="utrecht-form">
      <div className="utrecht-form-field">
        <p className="utrecht-form-label">Voornaam</p>
        <input
          type="text"
          name="first_name"
          className="utrecht-textbox utrecht-textbox--html-input utrecht-form-field__input"
        />
      </div>
    </form>
  </Story>
</Canvas>

## Fout: Label zonder gekoppelde input

Door het `<label>` element te gebruiken zonder `htmlFor` mis je de koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

## Fout: Input ID niet uniek

Door het id van de input dubbel te gebruiken krijg je niet de juiste koppeling tussen het label en de input voor mensen die de visuele context niet waarnemen.

          </label>
        </p>

          type="text"

          id="name"

        <p>
